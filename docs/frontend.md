# Frontend

## Overview

The frontend is a React 19 single-page application built with Vite. It uses
TanStack Router for file-based routing and Apollo Client for GraphQL data
fetching.

- **Entry point**: `app/src/main.tsx`
- **Dev server**: `http://localhost:3000`
- **GraphQL endpoint**: `/graphql` (proxied by Vite to `http://localhost:3001`)

## Bootstrap (`main.tsx`)

```ts
const client = new ApolloClient({
  cache: new InMemoryCache(),
  dataMasking: true,
  link: new HttpLink({ uri: "/graphql" }),
});

const router = createRouter({ routeTree });

ReactDOM.createRoot(document.getElementById("root")!).render(
  <React.StrictMode>
    <ApolloProvider client={client}>
      <RouterProvider router={router} />
    </ApolloProvider>
  </React.StrictMode>,
);
```

`dataMasking: true` means Apollo fragments are masked вЂ” components must use
`useFragment` to read fragment data from the cache.

## Routing

Routes are file-based via TanStack Router. The router config is
auto-generated into `routeTree.gen.ts` вЂ” **never edit this file by hand**.

To add a new route:
1. Create a file under `app/src/routes/` (e.g. `routes/persons/[id].tsx`).
2. Export `const Route = createFileRoute("/persons/$id")({ ... })`.
3. The router will pick it up and regenerate `routeTree.gen.ts` automatically
   during `npm run dev`.

The root layout (`__root.tsx`) renders the nav header and `<Outlet />` for
all child routes.

## Data Fetching Pattern

All GraphQL operations are co-located in the route or component file that
uses them. Use `graphql()` from `@/__generated__/gql.js` to define typed
documents:

```ts
import { graphql } from "@/__generated__/gql.js";

const GET_PERSONS = graphql(`
  query GetPersons {
    persons {
      id
      ...Person_List
    }
  }
`);
```

Then use Apollo hooks:

```ts
const { data, loading, error } = useQuery(GET_PERSONS);
const [createPerson] = useMutation(CREATE_PERSON, {
  refetchQueries: [{ query: GET_PERSONS }],
});
```

Always pass `refetchQueries` on mutations that modify lists.

## Fragment Pattern

List components define their own fragment and use `useFragment` to read data:

```ts
const PERSON_LIST = graphql(`
  fragment Person_List on Person {
    id
    firstName
    lastName
    email
  }
`);

function PersonRow({ person: from, onClickDelete }) {
  const { data: person, complete } = useFragment({
    fragment: PERSON_LIST,
    from,
  });

  if (!complete) return <Spinner />;
  // render ...
}
```

The parent query fetches `id` + spreads the fragment; the row reads from cache
via `useFragment`. This is the Apollo data masking / colocation pattern.

## Form Presentation Rule

**All forms must be presented inside a `Dialog`.** Never render a form inline
on a page or expand it in-place (e.g. toggling a section open). Always open a
`Dialog` with a clear title and a cancel button. This keeps the UI consistent
and avoids layout shift.

```tsx
// Correct вЂ” form in a Dialog
<Dialog open={open} onOpenChange={setOpen}>
  <DialogContent className="max-w-md">
    <DialogHeader>
      <DialogTitle>Add Note</DialogTitle>
    </DialogHeader>
    <NoteForm onSubmit={handleSubmit} onCancel={() => setOpen(false)} />
  </DialogContent>
</Dialog>

// Wrong вЂ” inline form
{showForm && <NoteForm onSubmit={handleSubmit} onCancel={() => setShowForm(false)} />}
```

## Form Pattern

Forms use TanStack Form with Zod validation:

```ts
const schema = z.object({ name: z.string().min(1, "Required") });

const form = useAppForm({
  defaultValues: { name: "" },
  validators: { onSubmit: schema },
  onSubmit: async ({ value }) => { /* call mutation */ },
});
```

`useAppForm` is created once per feature file via `createFormHook` and
re-exported. Field components (`TextField`) wrap `@/components/ui/form-field`.

## UI Components

`app/src/components/ui/` contains shadcn/ui primitives. These follow the
standard shadcn pattern:

- `forwardRef` for DOM forwarding
- `cn()` from `@/lib/utils` for class merging
- Variant props via `cva` (if present)

Do **not** put application logic in `ui/`. Custom components belong in
`components/domain/` or `components/layouts/`.

## Styling

Tailwind CSS v3 with the shadcn/ui preset. Config is at
`app/tailwind.config.ts`. Use Tailwind utility classes directly in JSX.
Use `cn()` when merging conditional class strings.

## Running & Building

```bash
npm run dev:app        # Vite dev server (port 3000)
npm run build:app      # Production build в†’ app/dist/
```

## Codegen

After changing any GraphQL query, mutation, or fragment in `app/src/`:

```bash
npm run codegen:app    # Regenerates app/src/__generated__/
```

The generated `graphql.ts` contains all TypeScript types; `gql.ts` provides
the typed `graphql()` template tag. Never edit `__generated__/` files by hand.
