# Database

## Technology

- **ORM**: [Drizzle ORM](https://orm.drizzle.team/) (`drizzle-orm/pg-core`)
- **Engine**: PGlite — embedded Postgres via WASM, no external server required
- **Schema file**: `db/src/schema.ts`
- **Relations file**: `db/src/relations.ts`
- **Connection**: `db/src/index.ts` (singleton, initialized at import time)
- **Migrations**: `db/drizzle/` (generated by `drizzle-kit generate`)

## Connection

`db/src/index.ts` creates a single PGlite instance at module load time:

```ts
const client = new PGlite(process.env.DATABASE_URL ?? "../pgdata");
await client.waitReady;
export const db = drizzle({ client, schema, relations });
export type DB = typeof db;
export { schema };
export * from "./schema.ts";
```

Server code imports from the `@philotes/db` package alias:

```ts
import { db } from "@philotes/db";
import type { DB } from "@philotes/db";
```

For tests, pass a fresh in-memory path (no `dataDir`) to isolate each run.

## Schema Tables

### `persons`

Primary entity. Represents a person in the CRM.

| Column       | Type          | Notes                  |
| ------------ | ------------- | ---------------------- |
| `id`         | `uuid` PK     | `defaultRandom()`      |
| `first_name` | `text`        | required               |
| `last_name`  | `text`        | required               |
| `email`      | `text`        | required, unique       |
| `created_at` | `timestamptz` | `defaultNow()`         |
| `updated_at` | `timestamptz` | `defaultNow()`         |
| `birthdate`  | `date`        | nullable               |

### `notes`

Free-form notes optionally linked to a person.

| Column      | Type      | Notes                           |
| ----------- | --------- | ------------------------------- |
| `id`        | `uuid` PK | `defaultRandom()`               |
| `body`      | `text`    | required                        |
| `person_id` | `uuid` FK | nullable → `persons.id`         |

### `labels`

Tags that can be applied to persons.

| Column  | Type      | Notes             |
| ------- | --------- | ----------------- |
| `id`    | `uuid` PK | `defaultRandom()` |
| `color` | `text`    | required (hex)    |
| `label` | `text`    | required          |

### `person_labels`

Many-to-many join between persons and labels.

| Column      | Type   | Notes                                       |
| ----------- | ------ | ------------------------------------------- |
| `person_id` | `uuid` | FK → `persons.id` ON DELETE CASCADE, PK part |
| `label_id`  | `uuid` | FK → `labels.id` ON DELETE CASCADE, PK part  |

Composite primary key: `(person_id, label_id)`.

### `important_dates`

Named dates (birthdays, anniversaries, deadlines, etc.) linked to a person.

| Column        | Type      | Notes                               |
| ------------- | --------- | ----------------------------------- |
| `id`          | `uuid` PK | `defaultRandom()`                   |
| `person_id`   | `uuid` FK | required → `persons.id` ON DELETE CASCADE |
| `name`        | `text`    | required (e.g. "Birthday")          |
| `description` | `text`    | nullable                            |
| `date`        | `date`    | required                            |

## Relations (`db/src/relations.ts`)

Defined with `defineRelations` from `drizzle-orm`:

```
persons
  ├── notes          (one-to-many)
  ├── labels         (many-to-many via person_labels)
  └── importantDates (one-to-many)

notes
  └── person         (many-to-one → persons)

importantDates
  └── person         (many-to-one → persons)
```

## Table Definition Pattern

```ts
import { pgTable, text, timestamp, uuid } from "drizzle-orm/pg-core";

export const things = pgTable("things", {
  id: uuid("id").primaryKey().defaultRandom(),
  name: text("name").notNull(),
  createdAt: timestamp("created_at", { withTimezone: true })
    .notNull()
    .defaultNow(),
  updatedAt: timestamp("updated_at", { withTimezone: true })
    .notNull()
    .defaultNow(),
  ownerId: uuid("owner_id").references(() => owners.id),
});

export type Thing = typeof things.$inferSelect;
export type NewThing = typeof things.$inferInsert;
```

### Column ordering convention

1. `id` — primary key, always first
2. Domain / business columns
3. `createdAt` / `updatedAt` timestamps (omit on simple join tables)
4. Foreign key columns

## Column Types

| Use case           | Drizzle helper                                  |
| ------------------ | ----------------------------------------------- |
| Primary key (UUID) | `uuid("id").primaryKey().defaultRandom()`        |
| Text               | `text("col_name")`                               |
| Integer            | `integer("col_name")`                            |
| Timestamp (TZ)     | `timestamp("col_name", { withTimezone: true })`  |
| Date               | `date("col_name")`                               |
| FK (UUID)          | `uuid("col_name").references(() => other.id)`    |

## Naming Conventions

| Item           | Convention           | Example                            |
| -------------- | -------------------- | ---------------------------------- |
| Table variable | `camelCase` plural   | `persons`, `importantDates`        |
| SQL table name | `snake_case` plural  | `"persons"`, `"important_dates"`   |
| Column var     | `camelCase`          | `firstName`, `personId`            |
| SQL column     | `snake_case`         | `"first_name"`, `"person_id"`      |

## Constraints

- **Primary key**: always `uuid("id").primaryKey().defaultRandom()` — do not use `serial()`.
- **Not-null**: mark required columns `.notNull()`. Leave optional columns without it.
- **Unique**: use `.unique()` where applicable (e.g. `email`).
- **Cascade deletes**: join tables and child records should use `{ onDelete: "cascade" }` on their FK.

## Type Exports

Export inferred types for every table directly below its definition:

```ts
export type Person = typeof persons.$inferSelect;
export type NewPerson = typeof persons.$inferInsert;
```

These are re-exported from `db/src/index.ts` via `export * from "./schema.ts"` and importable from server as:

```ts
import type { Person, NewPerson } from "@philotes/db";
```

## Migration Workflow

```bash
npm run db:generate   # Generates a new migration in db/drizzle/
npm run db:migrate    # Applies pending migrations
npm run db:push       # Dev shortcut: push schema without migration files
npm run db:studio     # Open Drizzle Studio GUI
```

Always run `db:generate` after editing `db/src/schema.ts`. Commit the generated migration file alongside your schema change.
